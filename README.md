# 🧠 Customer Feedback Intelligence Platform

> **Portfolio Project** — An end-to-end AI-powered customer feedback system that automatically classifies sentiment, generates personalized responses via LLM + RAG, visualizes data in Tableau, and automates reporting via n8n.

---

## 📌 Background & Problem Statement

In modern businesses, the volume of customer feedback keeps growing but often goes unprocessed systematically. Operations teams spend significant time:

- **Reading & classifying** feedback manually (positive / negative / neutral)
- **Responding** one by one without adequate historical context
- **Reporting** customer satisfaction reactively, not proactively
- **No early warning system** when there is a surge in negative feedback

This platform provides an end-to-end solution that automates the entire process.

---

## 🎯 Goals & Benefits

| Goal | Benefit |
|------|---------|
| Automatic sentiment classification | No need to manually read hundreds of feedbacks |
| Personalized AI response per feedback | Customers receive relevant, context-aware responses |
| RAG-powered context retrieval | Responses informed by similar past feedbacks |
| Automated daily reporting via n8n | Reports sent automatically to the team |
| Real-time visualization in Tableau | Data-driven decision making |

## 🔄 Full Workflow (Dataset → AI → Output)

### Step 1 — Input Dataset via test.sh

Test data is submitted using a shell script `test.sh` that sends 15 feedbacks via HTTP POST:

- **11 feedbacks** submitted on **May 1, 2025**
- **4 feedbacks** submitted on **May 2, 2025**

Each request contains:
```json
{
  "text": "Pengiriman sangat lambat dan barang datang dalam kondisi rusak",
  "customer_id": "cust_123",
  "product_id": "prod_456",
  "source": "website",
  "rating": 2,
  "category": "delivery"
}
```

> **Note:** `sentiment` and `sentiment_respon` are **NOT filled manually** — both are auto-generated by Gemini AI. `category` is provided by the user.

---

### Step 2 — Validation & Routing (FastAPI)

Requests arrive at `POST /api/v1/feedback`:
- Pydantic validates the schema (`FeedbackCreate`)
- Logging middleware records every request

---

### Step 3 — RAG Context Retrieval

Before calling Gemini, the system queries **Qdrant** for semantically similar past feedbacks:

```
rag.similarity_search(feedback.text, top_k=5)
    └── Returns up to 5 most similar past feedbacks (cosine similarity ≥ 0.3)
    └── Formatted into context string:
        "Similar past feedbacks:
         1. [NEGATIVE] Pengiriman terlambat 3 hari...
         2. [NEGATIVE] Barang sampai dalam kondisi rusak..."
```

This context is passed directly into Gemini's prompt so the AI response is grounded in real historical cases — not just the current feedback in isolation.

---

### Step 4 — Gemini AI Analysis

`GeminiService.analyze_feedback(text, rag_context)` sends a prompt to **Gemini 2.5 Flash** that includes the feedback text **and** the RAG context retrieved in Step 3.

Gemini returns exactly two fields:

| Field | Example Output |
|-------|---------------|
| `sentiment` | `negative` |
| `sentiment_respon` | "Kami sangat menyesal mendengar pengalaman buruk Anda. Kami akan segera menghubungi tim logistik untuk menindaklanjuti masalah keterlambatan dan kerusakan barang ini." |

Key behaviors:
- **`sentiment_respon` is personalized**, not a template — it directly addresses the specific issue mentioned in the feedback
- **Response language follows the feedback language** — Indonesian feedback → Indonesian response, English feedback → English response
- For **negative** feedback: sincere apology + concrete solution
- For **neutral** feedback: polite acknowledgment referencing specific content
- For **positive** feedback: warm thank-you echoing the positive aspects mentioned
- **Fallback** if Gemini fails: `sentiment_respon` defaults to `'Terima kasih atas ulasan Anda. Tim kami akan segera menindaklanjuti.'`

---

### Step 5 — Persist to PostgreSQL

Results are saved to the `feedback` table:

```
text, customer_id, product_id, source, rating, category  ← from user
sentiment, sentiment_respon                               ← from Gemini
created_at                                                ← auto
```

---

### Step 6 — Index to Qdrant (RAG Update)

After `db.flush()` returns the new `feedback.id`, the feedback is indexed into Qdrant so it becomes available as RAG context for **future** feedbacks:

```
rag.index_feedback(id, text, {sentiment, category, source, customer_id})
    └── Embed text → 384-dim vector
    └── Upsert to Qdrant collection "feedback_embeddings"
```

Then `db.commit()` finalizes the transaction.

---

### Step 7 — Output: n8n & Tableau

Data stored in PostgreSQL is consumed by:

**n8n (Automation)**:
- **Workflow 1** — Alert when new negative feedbacks appear → email to Ops Team
- **Workflow 2** — Automated daily report → Google Sheets + Email

**Tableau (Visualization)**:
- Customer Feedback Overview dashboard
- Category Distribution by sentiment
- Rating vs Sentiment Score scatter plot
- CSAT Score & AVG Rating KPIs
