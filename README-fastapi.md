# ⚡ FastAPI — Feedback Routes

Documentation for the feedback module of the Customer Feedback Intelligence Platform. This is the core of the system — handling feedback intake, RAG context retrieval, Gemini AI analysis, and data persistence.

---

## 🏁 Entry Point

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

On startup:
1. `init_db()` — creates all PostgreSQL tables if they don't exist
2. `RAGPipeline()` — initializes Qdrant connection and loads embedding model

Interactive API docs: `http://localhost:8000/docs`

---

## 🔄 Feedback Creation Flow

```
POST /api/v1/feedback
        │
        ▼
[1] Pydantic validation (FeedbackCreate)
        │
        ▼
[2] RAGPipeline.similarity_search(text, top_k=5)
    └── Retrieve up to 5 similar past feedbacks from Qdrant
    └── Build context string → rag_context
        │
        ▼
[3] GeminiService.analyze_feedback(text, rag_context)
    ├── Uses RAG context if available (SENTIMENT_WITH_CONTEXT_PROMPT)
    ├── Without context (SENTIMENT_ANALYSIS_PROMPT)
    └── Returns: { sentiment, sentiment_respon }
        │
        ▼
[4] Create Feedback object + db.flush()  → get new ID
        │
        ▼
[5] RAGPipeline.index_feedback(id, text, metadata)
    └── Embed + upsert to Qdrant
        │
        ▼
[6] db.commit() → data finalized
        │
        ▼
[7] Return FeedbackResponse
```

## 📋 Schemas

### FeedbackCreate (Request Body)

```python
class FeedbackCreate(BaseModel):
    text:        str
    customer_id: Optional[str] = None
    product_id:  Optional[str] = None
    source:      str = "web"
    rating:      Optional[int] = None
    category:    Optional[str] = None   # filled by user (delivery, price, etc.)
```

> `sentiment` and `sentiment_respon` are **not in the request** — both auto-generated by Gemini.

### FeedbackResponse

```python
class FeedbackResponse(BaseModel):
    id:               int
    text:             str
    customer_id:      Optional[str]
    product_id:       Optional[str]
    sentiment:        str              # ← AUTO: positive / neutral / negative
    sentiment_respon: Optional[str]   # ← AUTO: personalized AI response
    category:         Optional[str]   # ← from user
    rating:           Optional[int]
    source:           Optional[str]
    created_at:       datetime
```

## 🛣️ Endpoints

### `POST /api/v1/feedback`
Create feedback with full AI analysis.

```bash
curl -X POST http://localhost:8000/api/v1/feedback \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Pengiriman sangat lambat dan barang datang dalam kondisi rusak",
    "customer_id": "cust_123",
    "product_id": "prod_456",
    "source": "website",
    "rating": 2,
    "category": "delivery"
  }'
```

Response `201 Created`:
```json
{
  "id": 1,
  "text": "Pengiriman sangat lambat dan barang datang dalam kondisi rusak",
  "sentiment": "negative",
  "sentiment_respon": "Kami sangat menyesal mendengar pengalaman buruk Anda. Kami akan segera menghubungi tim logistik untuk menindaklanjuti masalah keterlambatan dan kerusakan barang ini.",
  "category": "delivery",
  "rating": 2,
  "source": "website",
  "created_at": "2025-05-01T10:00:00"
}
```

### `GET /api/v1/feedback`
List feedbacks with filters.

| Query Param | Default | Description |
|-------------|---------|-------------|
| `skip` | 0 | Pagination offset |
| `limit` | 10 | Max results (max 100) |
| `sentiment` | — | `positive` / `neutral` / `negative` |
| `category` | — | Filter by category |
| `days` | 7 | Feedbacks from last N days |

---

### `GET /api/v1/feedback/{id}`
Get single feedback by ID.

---

### `PUT /api/v1/feedback/{id}`
Manual override. Updatable fields: `text`, `category`, `rating`, `is_verified`.

---

### `DELETE /api/v1/feedback/{id}`
Delete feedback and remove its vector from Qdrant.

---

### `GET /api/v1/feedback/search?query=...`
Semantic search via RAG. Returns feedbacks sorted by cosine similarity.

| Param | Default |
|-------|---------|
| `query` | required |
| `top_k` | 5 (max 20) |

---

### `GET /api/v1/feedback/stats/summary`
Aggregated statistics. Used by n8n Workflow 2.

| Param | Description |
|-------|-------------|
| `days` | Last N days (default 7) |
| `date` | Specific date `YYYY-MM-DD` |

```json
{
  "total_feedbacks": 15,
  "sentiment_distribution": {"positive": 6, "neutral": 3, "negative": 6},
  "category_distribution": {"delivery": 3, "product_quality": 4},
  "average_rating": 3.0
}
```

---

### `GET /api/v1/feedback/report/export`
Flat export for n8n & Tableau. Each feedback row contains:
`id, customer_id, product_id, rating, text, sentiment, sentiment_respon, category, source, created_at`

| Param | Description |
|-------|-------------|
| `days` | Last N days (default 7) |
| `date` | Specific date `YYYY-MM-DD` |
| `sentiment` | Filter by sentiment |
| `category` | Filter by category |

## 🤖 Gemini AI — What It Does

### With RAG context available
Uses `SENTIMENT_WITH_CONTEXT_PROMPT` — Gemini sees similar past feedbacks and generates a response that is consistent with how similar issues were handled before.

### Without RAG context
Uses `SENTIMENT_ANALYSIS_PROMPT` — Gemini analyzes the feedback text alone and generates a personalized response based solely on its content.

### Output always:
```json
{
  "sentiment": "negative",
  "sentiment_respon": "Kami sangat menyesal atas pengalaman buruk Anda..."
}
```

- Response is **personalized**, not a template
- Response **language matches the feedback language**
- Negative → sincere apology + concrete solution
- Neutral → polite acknowledgment of specific content
- Positive → warm thank-you echoing what was praised

### Fallback (if Gemini fails):
```python
{
  "sentiment": <rule-based: positive/neutral/negative>,
  "sentiment_respon": "Terima kasih atas ulasan Anda. Tim kami akan segera menindaklanjuti."
}
```

## ⚠️ Error Handling

| Code | Condition |
|------|-----------|
| `201` | Feedback created successfully |
| `400` | Invalid sentiment value |
| `404` | Feedback ID not found |
| `500` | Gemini, DB, or RAG error → `db.rollback()` called |
